## 1. 개요

VMS Labeling 기능은 OCR 인식률 향상을 위해 **자체 학습 데이터를 구축**하고, **PaddleOCR 모델을 Fine-tuning**하여 현장 환경에 최적화된 커스텀 모델을 만드는 도구입니다.

### 전체 흐름

```
이미지 수집 → 데이터셋 생성 → 라벨링 → 내보내기 → 학습 → 커스텀 모델 적용
```

| 단계 | 설명 |
|------|------|
| 라벨링 | 이미지에서 텍스트 영역을 박스로 지정하고 실제 문자열을 입력 |
| 내보내기 | PaddleOCR 학습 포맷으로 데이터를 변환 |
| 학습 | Python 스크립트를 통해 PaddleOCR 모델을 Fine-tuning |
| 적용 | 학습된 ONNX 모델을 OCR Tool에 즉시 적용 |

## 2. 사전 준비

### 2.1 Python 환경 설치

학습 기능을 사용하려면 Python 환경이 필요합니다. (라벨링만 사용할 경우 불필요)

```bash
# Python 3.8 ~ 3.10 권장
pip install paddlepaddle paddleocr paddle2onnx onnx
```

> **참고**: GPU 학습을 위해서는 CUDA 지원 PaddlePaddle을 설치하세요.
> ```bash
> pip install paddlepaddle-gpu
> ```

### 2.2 학습 스크립트 위치

학습 스크립트는 VMS 설치 경로에 포함되어 있습니다:

```
VMS.VisionSetup/scripts/train_ppocr.py
```

## 3. 라벨링 화면 구성

VMS 메인 화면 상단의 **"Labeling"** 탭을 클릭하면 라벨링 화면으로 전환됩니다.

```
┌─────────────┬──────────────────────────┬──────────────┐
│  좌측 패널   │       중앙 캔버스         │  우측 패널    │
│             │                          │              │
│ [Dataset]   │                          │ [Classes]    │
│  - 목록     │     이미지 표시 영역       │  - 클래스 목록│
│  - 생성/삭제 │     + 바운딩 박스 편집     │  - 추가      │
│             │                          │              │
│ [Images]    │                          │ [Labels]     │
│  - 이미지   │                          │  - 라벨 목록  │
│    목록     │                          │  - 편집기     │
│  - 추가/삭제 │                          │              │
│  - ◀ ▶ 탐색 │                          │ [Export &    │
│             │                          │  Training]   │
└─────────────┴──────────────────────────┴──────────────┘
```

| 영역 | 기능 |
|------|------|
| **좌측 패널** | 데이터셋 관리, 이미지 목록 및 탐색 |
| **중앙 캔버스** | 이미지 표시, 바운딩 박스 생성/편집/삭제 |
| **우측 패널** | 클래스 관리, 라벨 목록, 라벨 편집, 내보내기 & 학습 |

## 4. 데이터셋 관리

### 4.1 데이터셋 생성

1. 좌측 패널의 **Dataset** 섹션에서 텍스트 입력란에 데이터셋 이름을 입력합니다.
   - 예: `VASIM_Label`, `ProductCode_2026`
2. **"+"** 버튼을 클릭하면 데이터셋이 생성됩니다.

> 데이터셋은 `%APPDATA%/VMS/Datasets/{데이터셋명}/` 폴더에 저장됩니다.

### 4.2 데이터셋 불러오기

- 목록에서 데이터셋을 선택한 후 **"Load"** 버튼을 클릭합니다.
- 저장된 이미지와 라벨 정보가 모두 로드됩니다.

### 4.3 데이터셋 저장

- **"Save"** 버튼을 클릭하면 현재 작업 내용이 JSON 파일로 저장됩니다.
- 이미지 전환 시 자동 저장되지만, 작업 중간에 수동 저장을 권장합니다.

### 4.4 데이터셋 삭제

- **"Delete"** 버튼 클릭 → 확인 대화 상자에서 승인하면 **모든 이미지와 라벨이 영구 삭제**됩니다.

> **주의**: 삭제된 데이터셋은 복구할 수 없습니다.

## 5. 이미지 추가 및 탐색

### 5.1 이미지 추가

1. 좌측 패널의 **Images** 영역에서 **"+ Add"** 버튼을 클릭합니다.
2. 파일 선택 대화 상자에서 이미지를 선택합니다.
   - 지원 포맷: BMP, JPG, JPEG, PNG, TIF, TIFF
3. 이미지가 데이터셋 폴더 안의 `images/` 하위 폴더에 복사됩니다.

> **팁**: 학습 효과를 높이려면 **실제 검사 환경에서 촬영한 이미지**를 충분히 수집하세요. 최소 50장 이상 권장합니다.

### 5.2 이미지 탐색

| 조작 | 설명 |
|------|------|
| **◀ / ▶ 버튼** | 이전/다음 이미지로 이동 |
| **이미지 목록 클릭** | 해당 이미지로 바로 이동 |
| **더블 클릭** | 이미지 선택 및 표시 |

- 현재 위치는 `[현재번호 / 전체]` 형식으로 표시됩니다.
- 이미지 목록에서 **녹색 원(●)** = 라벨링 완료, **회색 원(●)** = 미라벨링

### 5.3 이미지 삭제

- **"- Del"** 버튼으로 현재 이미지를 삭제할 수 있습니다.

## 6. 라벨링 작업

### 6.1 클래스 등록

라벨링을 시작하기 전에 **인식할 텍스트 종류(클래스)**를 등록합니다.

1. 우측 패널의 **Classes** 섹션에서 클래스 이름을 입력합니다.
   - 예: `text`, `serial_number`, `date`, `lot_code`
2. **"+"** 버튼을 클릭하여 추가합니다.
3. 목록에서 현재 사용할 클래스를 **선택(클릭)** 합니다.

> 클래스마다 자동으로 고유 색상이 배정되어, 캔버스에서 클래스별 바운딩 박스 색상이 다르게 표시됩니다.

### 6.2 바운딩 박스 생성

1. 우측 패널에서 사용할 **클래스를 선택**합니다.
2. 중앙 캔버스의 **도구 모음(Toolbar)** 에서 사각형 ROI 도구를 선택합니다.
3. 이미지 위에서 텍스트 영역을 **드래그하여 박스를 그립니다**.
4. 박스가 생성되면 자동으로 선택한 클래스의 라벨이 생성됩니다.

### 6.3 텍스트 입력 (Transcription)

**Recognition 학습에 필수적인 단계입니다.**

1. 캔버스에서 바운딩 박스를 **클릭하여 선택**합니다.
2. 우측 패널 하단의 **Selected Label** 편집기가 활성화됩니다.
3. **Transcription** 입력란에 해당 박스 안의 **실제 텍스트를 정확히 입력**합니다.
   - 예: 박스 안에 "ABC-1234"가 보이면 → `ABC-1234` 입력

> **중요**: Transcription은 대소문자, 공백, 특수문자를 포함하여 **정확하게** 입력해야 합니다. 이 텍스트가 모델 학습의 정답(Ground Truth)이 됩니다.

### 6.4 라벨 수정

| 작업 | 방법 |
|------|------|
| **박스 이동** | 박스를 드래그하여 위치 변경 |
| **박스 크기 조절** | 모서리/변 핸들을 드래그하여 크기 변경 |
| **클래스 변경** | 라벨 선택 후 편집기의 Class 드롭다운에서 변경 |
| **텍스트 수정** | 라벨 선택 후 Transcription 입력란 수정 |
| **라벨 삭제** | 라벨 선택 후 Labels 영역의 "Delete" 버튼 클릭 |

### 6.5 라벨링 품질 가이드라인

학습 효과를 극대화하기 위한 라벨링 규칙:

| 항목 | 권장 사항 |
|------|----------|
| **박스 범위** | 텍스트에 딱 맞게 — 너무 크거나 작으면 학습 품질 저하 |
| **텍스트 단위** | 한 줄에 하나의 박스 (여러 줄을 하나의 박스로 묶지 마세요) |
| **일관성** | 같은 형식의 텍스트는 동일한 클래스로 분류 |
| **Transcription** | 이미지에 보이는 그대로 정확히 입력 |
| **최소 수량** | Detection: 100장+ / Recognition: 500장+ 권장 |

## 7. 데이터 내보내기 (Export)

라벨링이 완료되면 학습 데이터 형식으로 내보냅니다.

### 7.1 자동 분할 (Train/Validation)

1. 우측 패널 하단의 **Export & Training** 섹션을 펼칩니다.
2. **"Auto Split (Train/Val)"** 버튼을 클릭합니다.
3. 전체 이미지가 80:20 비율로 Train/Validation에 자동 분배됩니다.

> 분할 결과는 상태 바에 `자동 분할 완료: Train 80장, Val 20장` 형태로 표시됩니다.

### 7.2 PaddleOCR 형식 내보내기

**PaddleOCR Fine-tuning에 사용할 포맷입니다.**

1. **"Export PaddleOCR"** 버튼을 클릭합니다.
2. 저장할 폴더를 지정합니다.
3. 다음 파일들이 자동 생성됩니다:

```
출력폴더/
├── train_det.txt        # Detection 학습용 라벨 (이미지경로 + 좌표)
├── val_det.txt          # Detection 검증용 라벨
├── train_rec.txt        # Recognition 학습용 라벨 (이미지경로 + 텍스트)
├── val_rec.txt          # Recognition 검증용 라벨
├── train_crops/         # Recognition용 크롭 이미지
│   ├── img_0001.jpg
│   ├── img_0002.jpg
│   └── ...
└── images/              # Detection용 원본 이미지
    ├── train/
    └── val/
```

### 7.3 YOLO 형식 내보내기

객체 탐지 학습(YOLO 계열)에 사용할 포맷입니다.

1. **"Export YOLO"** 버튼을 클릭합니다.
2. 저장할 폴더를 지정합니다.
3. `data.yaml` 및 normalized xywh 라벨 파일이 생성됩니다.

## 8. 모델 학습 (Training)

### 8.1 학습 설정

우측 패널의 **Export & Training** 섹션을 펼치면 Training 설정이 나타납니다.

| 항목 | 설명 | 기본값 |
|------|------|--------|
| **Python** | Python 실행 경로 | `python` |
| **Script** | 학습 스크립트 파일 경로 | (수동 지정) |
| **Epochs** | 학습 반복 횟수 | `100` |
| **Batch** | 배치 크기 (GPU 메모리에 맞게 조절) | `8` |

### 8.2 학습 스크립트 설정

1. **Script** 항목 옆의 **"..."** 버튼을 클릭합니다.
2. 파일 선택 대화 상자에서 학습 스크립트를 선택합니다:

```
D:\projects\VMS-Solution\VMS.VisionSetup\scripts\train_ppocr.py
```

> **참고**: Python 경로가 시스템 PATH에 없는 경우, 전체 경로를 입력합니다:
> ```
> C:\Python310\python.exe
> ```
> 또는 Anaconda 환경:
> ```
> C:\Users\{사용자}\anaconda3\python.exe
> ```

### 8.3 학습 실행

1. 데이터셋을 선택하고, PaddleOCR 형식으로 **Export를 먼저 완료**합니다.
2. 학습 설정을 확인합니다.
3. **"Start Training"** 버튼을 클릭합니다.

학습이 시작되면:

- **프로그레스 바**에 진행률이 표시됩니다.
- **Loss**: 손실값 (낮을수록 좋음)
- **Acc**: 정확도 (높을수록 좋음)
- **Log 영역**에 학습 로그가 실시간으로 표시됩니다.

```
[EPOCH] 1/100
[LOSS] 2.3456
[ACC] 0.1200
[PROGRESS] 1.0
[EPOCH] 2/100
[LOSS] 1.8901
[ACC] 0.3450
...
[ONNX] C:\output\model.onnx
[DONE]
```

### 8.4 학습 중지

- **"Stop"** 버튼을 클릭하면 학습이 중단됩니다.
- 중단 시점까지의 체크포인트가 저장됩니다.

### 8.5 학습 완료

학습이 정상 완료되면:

1. 자동으로 ONNX 포맷으로 변환됩니다.
2. 완료 대화 상자에 **ONNX 모델 파일 경로**가 표시됩니다.
3. 이 경로를 메모해 두세요 — 다음 단계에서 사용합니다.

### 8.6 학습 파라미터 조정 가이드

| 상황 | 조치 |
|------|------|
| Loss가 줄지 않음 | Learning Rate 낮추기 (0.0001), 데이터 더 수집 |
| Acc가 낮음 | Epochs 늘리기, Transcription 오류 점검 |
| GPU 메모리 부족 | Batch Size 줄이기 (4 또는 2) |
| 과적합 (Train Acc 높고 Val Acc 낮음) | 데이터 더 수집, Epochs 줄이기 |

## 9. 커스텀 모델 적용

학습이 완료된 ONNX 모델을 OCR Tool에 적용합니다.

### 9.1 모델 경로 설정

1. VMS 메인 화면에서 **OCR Tool**을 선택합니다.
2. 우측의 Tool Settings에서 Engine을 **PP-OCR ONNX**로 설정합니다.
3. 다음 항목에 학습된 모델 경로를 입력합니다:

| 항목 | 설명 | 예시 |
|------|------|------|
| **Custom Det Model** | Detection ONNX 파일 경로 | `C:\output\det_model.onnx` |
| **Custom Rec Model** | Recognition ONNX 파일 경로 | `C:\output\rec_model.onnx` |
| **Custom Dict File** | 문자 사전 파일 경로 | `C:\output\dict.txt` |

> 세 항목 모두 입력할 필요는 없습니다. **학습한 모델만 지정**하면 됩니다.
> - Detection만 학습한 경우 → Custom Det Model만 입력
> - Recognition만 학습한 경우 → Custom Rec Model + Custom Dict File 입력
> - 나머지는 기본 PP-OCRv4 모델이 사용됩니다.

### 9.2 적용 확인

- 모델 경로가 하나라도 설정되면 **"Custom model active"** (녹색) 표시가 나타납니다.
- 이미지를 실행하여 인식 결과를 확인합니다.

### 9.3 기본 모델로 복원

- Custom Det Model, Custom Rec Model, Custom Dict File의 **내용을 모두 지우면** 기본 PP-OCRv4 모델로 자동 복원됩니다.

## 10. 전체 워크플로우 요약

```
┌──────────────────────────────────────────────────────────────┐
│                    VMS 딥러닝 학습 워크플로우                    │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  Step 1. 데이터셋 생성                                        │
│    └─ Labeling 탭 → Dataset → 이름 입력 → "+" 클릭            │
│                                                              │
│  Step 2. 이미지 추가                                          │
│    └─ "+ Add" → 검사 환경 이미지 선택 (50장 이상 권장)           │
│                                                              │
│  Step 3. 클래스 등록                                          │
│    └─ Classes → 텍스트 종류별 클래스명 입력 → "+" 클릭           │
│                                                              │
│  Step 4. 라벨링                                               │
│    └─ 각 이미지에서:                                           │
│       1) 클래스 선택                                           │
│       2) 텍스트 영역에 바운딩 박스 그리기                        │
│       3) Transcription에 실제 텍스트 입력                      │
│       4) ◀/▶ 로 다음 이미지                                   │
│                                                              │
│  Step 5. 내보내기                                             │
│    └─ "Auto Split" → "Export PaddleOCR"                      │
│                                                              │
│  Step 6. 학습                                                 │
│    └─ Script 경로 지정 → Epochs/Batch 설정 → "Start Training" │
│                                                              │

│    └─ OCR Tool Settings → Custom Model Path에 ONNX 경로 입력  │
│                                                              │
│  Step 8. 검증                                                 │
│    └─ 검사 이미지 실행 → 인식 결과 확인 → 필요 시 반복           │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 11. 문제 해결 (FAQ)

### Q: "학습 스크립트 경로를 지정하세요" 오류가 나옵니다.
**A**: Script 항목에 `train_ppocr.py` 파일 경로를 지정해야 합니다. "..." 버튼으로 `VMS.VisionSetup/scripts/train_ppocr.py`를 선택하세요.

### Q: 학습 시작 시 Python을 찾을 수 없다는 오류가 나옵니다.
**A**: Python 항목에 전체 경로를 입력하세요. 예: `C:\Python310\python.exe`. 터미널에서 `python --version`으로 설치 여부를 확인하세요.

### Q: `paddlepaddle` 모듈을 찾을 수 없다는 오류가 나옵니다.
**A**: Python 환경에 필요 패키지를 설치하세요:
```bash
pip install paddlepaddle paddleocr paddle2onnx onnx
```

### Q: GPU 메모리 부족 (CUDA Out of Memory) 오류가 발생합니다.
**A**: Batch Size를 줄여보세요 (8 → 4 → 2 → 1). GPU 메모리가 4GB 미만이면 CPU 학습을 권장합니다.

### Q: 학습은 완료했는데 인식률이 개선되지 않습니다.
**A**: 다음을 확인하세요:
- 라벨링 데이터가 충분한가? (Recognition: 최소 500장)
- Transcription이 정확한가?
- Train/Val 분할이 되어 있는가?
- Epochs를 늘려보세요 (200~500)

### Q: 기본 모델로 돌아가고 싶습니다.
**A**: OCR Tool Settings에서 Custom Det Model, Custom Rec Model, Custom Dict File을 모두 비우면 기본 PP-OCRv4 모델이 사용됩니다.

### Q: 이미지 목록에서 녹색/회색 원은 무엇인가요?
**A**: **녹색(●)** = 라벨이 1개 이상 있는 이미지, **회색(●)** = 아직 라벨이 없는 이미지입니다.

### Q: 데이터셋은 어디에 저장되나요?
**A**: `%APPDATA%/VMS/Datasets/` 폴더에 데이터셋별 하위 폴더로 저장됩니다. 각 데이터셋 폴더 안에 `dataset.json` (메타데이터)과 `images/` (이미지 파일)가 있습니다.
